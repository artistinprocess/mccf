# MCCF End-to-End Test Procedure

**Version:** 1.0  
**Date:** April 2026  
**System:** Multi-Channel Coherence Field v1.7  
**Platform:** Windows 11, Python 3.14, Flask, Ollama (llama3.2)

This document is the complete test procedure for verifying all MCCF
subsystems operate as designed. Tests are ordered from foundational
to composite — each layer depends on the ones below it. A failure at
any step must be resolved before continuing.

Record pass/fail and any anomalous output for each step.

---

## Prerequisites

Before beginning any test:

1. Git Bash is open
2. Working directory is correct:
   ```
   cd /d/VideoRenders/FederatedDialog/mccf_github_release/mccf_full
   ```
3. Local files match GitHub:
   ```
   git pull origin master
   cp *.html static/
   ```
4. No Flask server is currently running (check with `ps aux | grep py`)

### T-12: Sensor Event — Forward Direction

In the editor Sensor panel, fire one event:
- From: TestSteward
- To: TestArchivist
- Channel values: E=0.7, B=0.5, P=0.6, S=0.5

**Verify:**
```
http://localhost:5000/field
```

**Expected:** `episode_count` is now 1.
Matrix shows a non-zero coherence value for TestSteward ? TestArchivist.

**Also verify the sensor response.** The POST to /sensor returns
affect parameters. In the editor these should update the right pane:
`approach_factor`, `arousal`, `valence`, `engagement`,
`regulation_state`, `coherence_to_other`.

**Pass criteria:** All six affect parameters present in response.
All values in their defined ranges:
- approach_factor: 0.0–1.0
- arousal: 0.0–1.0
- valence: -1.0–1.0
- engagement: 0.0–1.0
- regulation_state: 0.0–1.0
- coherence_to_other: 0.0–1.0

---

### T-13: Asymmetry Verification

Fire one sensor event in the reverse direction:
- From: TestArchivist
- To: TestSteward
- Channel values: E=0.3, B=0.5, P=0.4, S=0.6

**Verify:**
```
http://localhost:5000/field
```

**Expected:** Matrix shows TWO distinct values:
- TestSteward ? TestArchivist (from T-12)
- TestArchivist ? TestSteward (from this event)

These values should be DIFFERENT. R_ij ? R_ji is a fundamental
design property. If the values are identical the asymmetric coherence
record is broken.

**Pass criteria:** The two directional coherence scores differ by
at least 0.01.

---

### T-14: Coherence Accumulation

Fire 15 more sensor events from TestSteward to TestArchivist using
the same channel values as T-12.

**Verify after each set of 5:**
```
http://localhost:5000/field
```

**Expected:** The coherence score for TestSteward ? TestArchivist
increases toward 0.5–0.7 range over 15+ consistent episodes.
It should not jump immediately — the decay-weighted history
means early episodes have less influence than recent ones.

**Pass criteria:** Coherence score is measurably higher after 15
episodes than after 1 episode. Score does not exceed 1.0.

---

### T-15: Echo Chamber Risk Detection

Continue firing sensor events until both directions have 20+
episodes with very similar channel values (high mutual alignment).

**Verify:**
```
http://localhost:5000/field
```

Check `echo_chamber_risks` in the response.

**Expected:** The risk score for the TestSteward/TestArchivist pair
should be rising as mutual coherence increases without dissonance.

**Pass criteria:** echo_chamber_risk value is between 0.0 and 1.0.
High mutual alignment (both directions > 0.7) should produce risk
score above 0.5.

## FAILURE MODE TESTS

These tests deliberately stress the system to verify failure
detection mechanisms work.

### T-F1: Attractor Bifurcation Detection

Create two new agents: **Purist** and **Rebel**.
Configure Purist with high E/low B weights.
Configure Rebel with low E/high B weights.

Fire 10 sensor events: Purist ? TestSteward with E=0.9, B=0.1
Fire 10 sensor events: Rebel ? TestSteward with E=0.1, B=0.9

**Expected:** TestSteward identity drift report shows competing
drift vectors. echo_chamber_risk with Purist and Rebel should
diverge. drift_warning may trigger.

**Pass criteria:** System handles competing influences without
crashing. Drift report shows non-zero drift in both E and B
directions.

---

### T-F2: Window Collapse Simulation

Fire 25 sensor events in rapid succession from TestSteward
to TestArchivist with rapidly varying channel values
(alternate E=0.9,B=0.1 and E=0.1,B=0.9 each event).

**Expected:** coherence score oscillates rather than stabilizing.
The decay-weighted history prevents the system from locking onto
a false stable state.

**Pass criteria:** coherence score after 25 noisy events is
lower than after 15 consistent events in T-14. System remains
responsive. No 500 errors.

---

### T-F3: Over-Consolidation Check

Fire 50 identical sensor events from TestSteward to TestArchivist.

**Verify:**
```
http://localhost:5000/field
```

Check echo_chamber_risks.

**Expected:** echo_chamber_risk for the pair rises above 0.8.

Then fire one dissonant event: E=0.1, B=0.9, P=0.1, S=0.9.

**Expected:** coherence_delta is negative (dissonance detected).
System does not crash or produce invalid values.

**Pass criteria:** echo_chamber_risk correctly identifies
over-consolidation. Dissonant event produces measurable response.
