## Priority 3 — Mathematical Improvements (V2.2 field quality)

### mccf_hotHouse.py

- [ ] **BUG:** Clip T_ij to [0,1] after each TrustField update step.
      Grok fixed point analysis: T* = 2.5(1-d), exceeds [0,1] when d≈0.
      Fix: `T_ij = max(0.0, min(1.0, T_ij))` in TrustField.update()
- [ ] Document time constant 1/β = 50 time units in config reference.
      Add β and γ to CONFIGURATION_REFERENCE as tunable parameters.
- [ ] Verify ψ clamping range — current implementation clips to [0,1].
      Gemini suggested [-1,1]. Confirm which range is correct per the
      mathematical theory (channel weights are probability-like [0,1]).
      Document the decision explicitly.
- [ ] Add damping coefficient to Euler integrator — friction term that
      increases as agent approaches waypoint attractor, reduces "jitter"
      at high-pressure waypoints
- [ ] Add Hysteresis to TrustField — make γ a function of historical
      minimum R_ij. If Rupture recorded in CoherenceRecord history:
      `gamma_effective = gamma * 2.0`
- [ ] Evaluate CCS multiplicative formulation: R_raw^σ vs current
      σ·R_raw + (1-σ)·0.5. Grok critique: convex form makes low-CCS
      agents too centrist; multiplicative form produces undifferentiated
      (decoupled) behavior which is more biologically accurate for
      vmPFC analog. Test both on arc export data before committing.

### mccf_voice_api.py

- [ ] Replace scalar sentiment broadcast with Semantic Decomposition Matrix:
      - future/plan/likely/predict → P channel nudge
      - we/us/together/shared → S channel nudge
      - feel/hurt/care/fear → E channel nudge
      - act/do/consistent/reliable → B channel nudge
- [ ] Add uncertainty markers to sentiment vocabulary:
      "I guess", "maybe", "perhaps", "not sure" → negative valence signal
      (prevents LLM politeness bias from blocking W5 Rupture)
- [ ] Make `/arc/record` non-blocking — return 202 Accepted with current
      ψ vector immediately, process sentiment+field update in background thread
      NOTE: arc export must read post-processing state. Implement
      per-arc completion event or lock.
